<objective>
Fix two issues that would cause failures specifically in Vite's production build (Rollup strict mode) and during fast user interaction before the module finishes loading.

Purpose:

1. `cw_runningInterval` — an implicit global in `src/app.js` (assigned without `var`). Rollup's strict-mode production build throws `ReferenceError: cw_runningInterval is not defined` in `toggleDisplay()`. Must add `var cw_runningInterval = null;` before first use.
2. Camera button timing — `<script type="module">` is deferred, so the DOM is visible and interactive before the IIFE runs. The `Watch Leader` button fires `cw_setCameraTarget(-1)` as an inline onclick. If clicked before the module loads, `cw_setCameraTarget is not defined`. Add `disabled` to the button; remove it in `cw_init()`.

Output: One-line addition to `src/app.js` near line 1479. One attribute added to `index.html` line 189. Three lines added to `cw_init()` in `src/app.js`.
</objective>
